# Animals — Simplified Ecosystem Simulation (Lazarus / Free Pascal)

**Animals** is a visual agent-based ecosystem simulation developed in **Lazarus / Free Pascal**. The project is integrated with the **CHATGPT** library (specifically using the simulation components of the `openai_simulation` package), demonstrating the practical application of cellular grids, evolutionary rules, and stable double-buffered cycle execution.

## Features

The simulation models the survival and reproduction dynamics of a simplified ecosystem in an $80 \times 80$ grid. The system is governed by 5 types of entities:

| Type | Color | Behavior |
|---|---|---|
| **Plant** | Green | Primary resource, reproduces after a few cycles if there is space. |
| **Herbivore** | Blue | Moves and feeds on plants. Dies of old age or starvation. |
| **Carnivore** | Red | Introduced at cycle 200, feeds on herbivores. Dies of old age or starvation. |
| **Organic Matter** | Brown | Generated by the death of plants, herbivores, or carnivores. Degrades after a few cycles. |
| **Bacteria** | Yellow | Spawns from organic matter degradation, consumes matter, and reproduces. |

## Design Principles & Stability

This version has been completely restructured to ensure stability and prevent memory issues (Access Violations and Double-Frees):

1. **Safe Double Buffering**: The grid (`TTabuleiro`) keeps `FBoard` (active state) and `FNextBoard` (next state). All changes are written to `FNextBoard` and committed at the end of each cycle.
2. **Single Ownership**: A living object (`TSer`) belongs to only one buffer at a time.
3. **Safe Memory Release**: Dying entities are marked with `Morto := True` and removed from the active simulation. The memory release (`Free`) is handled exclusively during the board's `Commit` phase.
4. **Fixed Seed**: Uses a fixed pseudo-random seed at initialization for scientific reproducibility.

## Integration with CHATGPT Library

The core class `TSer` (in `uTipos.pas`) inherits directly from `TAISimEntity` of the `openai_simulation` package. This allows the ecosystem to be used as a simulated environment for training AI models, logical agent decision-making, and ecological data analysis.

---

## Code Structure

* **`uTipos.pas`**: `TSer` class extending `TAISimEntity`.
* **`uTabuleiro.pas`**: Grid manager with double buffering and safe memory management.
* **`uConfig.pas`**: `TConfig` record with calibrated ecological parameters.
* **`uSimulacao.pas`**: Core engine that processes simulation steps and applies biological rules.
* **`uEstat.pas`**: Statistical recording of population count per cycle.
* **`form2.pas` / `form2.lfm`**: Visual control interface with Start, Pause, Stop, Restart buttons and CSV export.

## How to Build and Run

### Requirements
- Lazarus IDE
- Free Pascal Compiler (FPC)
- CHATGPT Library (folder `pacote` accessible in search paths defined in `.lpi`)

### Building
Open the terminal in the project directory and run:
```bash
lazbuild animal.lpi
```

Or open `animal.lpi` directly inside the Lazarus IDE and press **F9**.
